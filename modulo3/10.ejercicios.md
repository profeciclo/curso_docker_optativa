### Ejercicio 1: Servidor Web con Persistencia Dual (Bind Mount + Volume)

Quieres desplegar un servidor Apache para una página web. Necesitas que el código HTML se pueda editar directamente desde tu ordenador (host), pero quieres que los archivos de "logs" del servidor se guarden en un volumen gestionado por Docker para que no se pierdan si borras el contenedor.

1. Crea un directorio en tu máquina local llamado `mi_web` y dentro crea un archivo `index.html` con un **mensaje de bienvenida con tu nombre**.
2. Crea un volumen de Docker llamado `logs_apache`.
3. Lanza un contenedor de la imagen `httpd:2.4` con las siguientes características:
    * Nombre: `servidor_tu_nombre`.
    * Mapeo de puertos: `8181:80`.
    * **Bind Mount:** Monta tu directorio local `mi_web` en la ruta `/usr/local/apache2/htdocs` del contenedor.
    * **Volumen:** Monta el volumen `logs_apache` en la ruta de logs del servidor: `/usr/local/apache2/logs`.
4. Accede a `localhost:8181` y verifica que ves tu HTML. 
    * Modifica el `index.html` en tu ordenador y refresca el navegador.
    * Borra el contenedor (`docker rm -f`) y crea uno nuevo con el mismo volumen de logs. Comprueba con `docker exec` que los archivos de log anteriores siguen ahí.

### Ejercicio 2: Infraestructura Aislada para Base de Datos y Administración

Vas a desplegar una base de datos MariaDB y una herramienta web para gestionarla llamada **Adminer**. Ambos deben estar en una red privada para que no cualquiera pueda ver el tráfico de la base de datos, y solo Adminer debe ser accesible desde el exterior.

1. Crea una red de tipo bridge definida por el usuario llamada `red_datos`.
2. Despliega un contenedor de **MariaDB**:
    * Nombre: `db_tu_nombre`.
    * Red: `red_datos`.
    * Variable de entorno: Define una contraseña para el root (`MARIADB_ROOT_PASSWORD`).
    * Persistencia: Usa un volumen para la ruta `/var/lib/mysql`.
    * **Nota:** No mapees ningún puerto al host para este contenedor (manténlo aislado).
3. Despliega un contenedor de **Adminer** (imagen `adminer`):
    * Nombre: `gestor_web_tu_nombre`.
    * Red: `red_datos`.
    * Mapeo de puertos: `8080:8080`.
4. **Comprueba que funciona con:**
    * Entra en `localhost:8080` desde tu navegador.
    * En el formulario de login de Adminer, en el campo "Servidor", escribe el nombre del contenedor de la base de datos (`db_ejercicio`). 
    * Introduce las credenciales que configuraste y verifica que puedes entrar.
