# Introducción a contenedores

### 1. ¿Qué es la Virtualización?
Es una tecnología que utiliza software para simular hardware, permitiendo ejecutar varios sistemas y aplicaciones de forma independiente en un mismo servidor físico. Su objetivo principal es el **aprovechamiento máximo de los recursos** de hardware que normalmente se infrautilizan.



### 2. Ventajas e Inconvenientes
* **Ventajas:** Ahorro económico y energético, mayor seguridad, optimización de recursos y facilidad para mover sistemas (migración en vivo).
* **Inconvenientes:** Dependencia de un único equipo físico (si falla el host, fallan todos) y una ligera pérdida de rendimiento por la capa de software adicional.

### 3. Tipos de Virtualización
1.  **Emulación:** Simula una arquitectura completa (ej. QEMU).
2.  **Tipo 1 (Nativa):** El software corre directamente sobre el hardware (ej. KVM, VMware ESXi).
3.  **Tipo 2 (Hosted):** El software corre sobre un sistema operativo ya instalado (ej. VirtualBox).
4.  **Virtualización Ligera (Contenedores):** Virtualización a nivel de sistema operativo que aísla procesos sin necesidad de simular hardware completo.

### 4. Contenedores: Definición y Clasificación
Un contenedor es un conjunto de procesos aislados que comparten el núcleo del sistema operativo anfitrión (*host*), pero con su propio sistema de archivos y red.
* **De Sistema:** Funcionan como mini máquinas virtuales (ej. LXC).
* **De Aplicación:** Optimizados para ejecutar y desplegar servicios específicos o aplicaciones web (ej. Docker, Podman).

### 5. Contenedores en el Desarrollo Moderno
Son el entorno ideal para las **arquitecturas de microservicios**, donde cada componente de la aplicación se despliega de forma independiente y se comunica mediante APIs.

### 6. Beneficios Clave de los Contenedores
* **Portabilidad:** Funcionan igual en cualquier entorno (desarrollo, prueba o producción).
* **Aislamiento:** Evitan conflictos entre aplicaciones y dependencias.
* **Eficiencia:** Consumen menos recursos que las máquinas virtuales al compartir el núcleo del SO.
* **Velocidad:** Permiten desplegar y escalar aplicaciones en cuestión de segundos.
