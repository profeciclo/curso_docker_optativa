# 9. Ejercicios
Entrega un documento con capturas de pantalla de la terminal donde se vean los comandos ejecutados y, especialmente, el resultado de `docker ps` y la visualización del servidor web en el navegador para estas dos actividades.


**Importante:**

Donde aparezca `tu_nombre` cambiadlo por vuestro nombre.

---

### Actividad 1: Contenedores y Ciclo de Vida

1.  **Creación:** Crea un contenedor llamado `terminal_repaso_tu_nombre` utilizando la imagen de **Ubuntu**, pero hazlo de forma que no se inicie automáticamente (usa `docker create`).
2.  **Arranque e Interacción:** Inicia el contenedor y accede a él de forma interactiva. Una vez dentro, crea un archivo en la carpeta `/tmp` llamado `saludo.txt` que **contenga tu nombre**. Sal del contenedor.
3.  **Inspección:** Sin borrar el contenedor, utiliza el comando `docker inspect` para encontrar la siguiente información y anótala:
    * El ID completo del contenedor.
    * El estado actual (`Running: true` o `false`).
    * La fecha de creación.
4.  **Ejecución Externa:** Con el contenedor parado, vuelve a iniciarlo. Ahora, sin entrar en él (usando `docker exec`), ejecuta un comando que muestre el contenido del archivo `/tmp/saludo.txt` en tu terminal anfitriona.
5.  **Limpieza:** Para el contenedor y bórralo definitivamente. Intenta borrarlo "a la fuerza" (`-f`) mientras esté encendido para comprobar la diferencia.

---

### Actividad 2: Despliegue de un "Stack" Web Básico (Apache + MariaDB)

1.  **Servidor Web Personalizado:** Lanza un contenedor de **Apache (httpd:2.4)** en segundo plano llamado `web_clase_tu_nombre`.
    * Mapea el puerto **8085** de tu máquina local al puerto **80** del contenedor.
    * Entra en el contenedor y modifica el archivo `index.html` para que muestre el título "Bienvenido a mi servidor Docker" y **tu nombre**.
    * Verifica en el navegador de tu equipo que puedes ver la página accediendo a `localhost:8085`.
2.  **Base de Datos Segura:** * Lanza un contenedor de **MariaDB** llamado `db_clase` en segundo plano.
    * Configura mediante variables de entorno (`-e`) una contraseña para root: `password123` y crea una base de datos inicial llamada `proyecto_daw`.
    * Mapea el puerto **3306** del contenedor al **3307** de tu equipo (para evitar conflictos si ya tienes MariaDB instalado).
3.  **Monitorización y Logs:**
    * Accede a los logs del servidor Apache en tiempo real (`-f`). Mientras los ves, refresca la página en tu navegador y observa cómo se generan las líneas de log de acceso.
    * Utiliza `docker ps` para mostrar ambos contenedores funcionando y asegúrate de que la columna "PORTS" refleje correctamente los mapeos realizados.
